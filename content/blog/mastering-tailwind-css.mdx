## Why Utility-First Still Wins

Rather than abstracting styles into custom class names like `.card` or `.btn`, Tailwind encourages you to **compose utilities directly in your markup**, producing styles that are readable, responsive, and easy to manage.

Compare this:

```css
/* Traditional CSS */
.card {
  padding: 1.5rem;
  border-radius: 0.75rem;
  background-color: white;
  box-shadow: 0 4px 6px rgba(0, 0, 0, 0.1);
}
```

To this in Tailwind:

```html
<div class="rounded-xl bg-white p-6 shadow-md">
  <!-- Content -->
</div>
```

It looks verbose at first‚Äîbut the clarity and maintainability quickly win you over, especially in large-scale projects.

## Getting Started with Tailwind v4 (Zero Config!)

Tailwind v4 brings a zero-config developer experience. Here‚Äôs how to set it up in a modern framework like Next.js or Vite.

1. Install Tailwind

```bash
npm install -D tailwindcss
```

That‚Äôs it! No need to install PostCSS or Autoprefixer manually ‚Äî Tailwind v4 handles it internally.

2. Add Tailwind to Your CSS Entry Point

In your globals.css or styles.css:

```css

@tailwind components;
@tailwind utilities;
```

3. Done.
   Tailwind will now automatically scan your entire project for class names. You don‚Äôt even need a tailwind.config.js unless you want to extend or customize the theme.

## Core Concepts Refresher

### Responsive Utilities

Tailwind uses mobile-first breakpoints like sm:, md:, and lg::

```html
<div class="w-full md:w-1/2 lg:w-1/3">
  <!-- Responsive layout -->
</div>
```

### Pseudo-Class Support

Want hover or focus states? Just prefix utilities:

```html
<button class="rounded bg-blue-600 px-4 py-2 text-white hover:bg-blue-700">
  Submit
</button>
```

Tailwind also supports advanced states like group-hover, focus-visible, and disabled:.

### Dark Mode

Tailwind v4 defaults to media-based dark mode, but you can opt-in to class-based mode easily with a config:

```js
// tailwind.config.js (only if needed)
export default {
  darkMode: "class",
};
```

Then use:

```html
<div class="bg-white text-black dark:bg-neutral-900 dark:text-white">
  Dark mode ready üåí
</div>
```

## Customization (Optional But Powerful)

If the defaults don‚Äôt match your design system, just add a config file:

```bash
npx tailwindcss init
```

You can now extend colors, spacing, fonts, and more:

```js
// tailwind.config.js
export default {
  theme: {
    extend: {
      colors: {
        brand: "#ff6363",
      },
      fontFamily: {
        sans: ["Inter", "sans-serif"],
      },
    },
  },
};
```
